**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

Для соискателей:
Отсутствует единый, структурированный и проверенный источник информации о реальном опыте прохождения собеседований в различных компаниях.
Сложность в целевой подготовке к тестовым заданиям конкретных работодателей из-за дефицита практических материалов, соответствующих актуальным требованиям.
Затруднён поиск отзывов и аналитики о компаниях от реальных сотрудников и кандидатов, что мешает формированию полной картины работодателя.

Для компаний (работодателей):
Отсутствие удобного инструмента для проведения масштабных конкурсов или тестовых испытаний для отбора кандидатов на стажировки или вакансии.
Невозможность быстро получить аналитику и статистику по результатам проведённых испытаний для оценки эффективности отбора и уровня подготовки аудитории.

- Публикация статьи/отзыва

- Прохождение тренировочных тестовых заданий

- Экспорт результатов в различные форматы
- Админ панель для модерации сайта

**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

Анализ 3 программных продуктов, которые максимально приближены к заданному функционалу, показал, что:

- LeetCode: не обладает возможностью публикации полноценных конкурсов от лица компании, не обладает возможностью создавать личные разделы пользователей
- Reddit: отсутствие раздела с тренировочными тестовыми заданиями, отсутствие возможности создания конкурса и просмотра статистики по нему у компаний
- Lolzteam: отсутствие раздела с тренировочными тестовыми заданиями, отсутствие сортировки отзывов и статей по компаниям
- Хабр: отсутствие возможности создания конкурса и просмотра статистики по нему у компаний, не обладает возможностью создавать личные разделы пользователей
- LinkedIn: не обладает возможностью создавать личные разделы пользователей, не обладает возможностью публикации полноценных конкурсов от лица компании, заблокирован на территории РФ

**[ ВОЗМОЖНЫЕ РИСКИ ]**

Создание неудобного пользовательского интерфейса
Пути преодоления:
Использовать дизайн-системы и UI-библиотеки (например, Material-UI, Ant Design) как основу для интерфейса.
Провести серию A/B-тестирований ключевых пользовательских сценариев для оптимизации UX.
Регулярно собирать фидбэк от первых пользователей через встроенные формы обратной связи и опросы.

Неправильная оценка технологического стека и нехватка времени на изучение
Пути преодоления:
Провести детальный анализ технологий на этапе планирования, проконсультировавшись с опытными разработчиками.
Выбрать технологии с большим сообществом и хорошей документацией (React для фронтенда, Django/FastAPI для бэкенда, PostgreSQL для базы данных).

Ложность реализации качественной системы редактирования статей
Пути преодоления:
Использовать готовые проверенные решения (например, редакторы TinyMCE, Quill или CKEditor) вместо разработки с нуля.
Реализовать MVP с базовым функционалом форматирования текста, с последующим постепенным добавлением возможностей.
Провести пользовательское тестирование на ранних этапах для определения действительно необходимого функционала.
